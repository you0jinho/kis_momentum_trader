## 발견된 패턴 / 참고할 점

### 1. 주문 직후 잔고조회에 짧은 "반영 지연"이 있음
주문 자체는 항상 즉시 `rt_cd=0` + 정상 주문번호(ODNO)로 접수되지만, **그 직후
곧바로 잔고를 재조회하면 수량/금액이 한 번에 다 안 잡히고 1~2초 안에 순차적으로
맞춰지는 현상**이 반복적으로 보였습니다 (예: 41주 매수했는데 직후 조회는
0주 → 몇 초 뒤 재조회하면 정상적으로 41주). 이건 코드 버그가 아니라 **모의투자
매칭/정산 엔진이 비동기적으로 처리되는 데서 오는 정상적인 지연**으로 보입니다.
체결 자체는 100% 확인됐으니 걱정할 부분은 아니지만, "즉시 확인되는 숫자"가
실제 최종값이 아닐 수 있다는 점은 참고하세요.

### 2. 같은 종목이 같은 세션에서 여러 번 재진입함
알테오젠(196170)은 09:12 매수 → 09:14 수익청산 → **즉시 재매수** → 09:15
손실청산까지, 30분 세션 안에서 총 2번의 매수-매도 사이클을 거쳤습니다. 이건
"매도 후에도 현재가가 여전히 돌파 기준선 위에 있으면 곧바로 재진입"하는 현재
로직이 의도대로 작동한 결과입니다. 재진입을 막고 싶다면(예: 하루 한 종목당
1회만 진입), 별도로 요청해주시면 수정해드릴 수 있습니다.

### 3. 간헐적 500 에러는 여전히 빈번함
종목 간 0.5초 딜레이를 넣었음에도, 거의 모든 호출 사이클에서 1차 시도가 500
에러로 실패하고 재시도(2~3초 후)에서 성공하는 패턴이 계속 보였습니다. 재시도
로직이 잘 동작해서 전체 흐름은 끊기지 않았지만, 이건 모의투자 서버 자체의
불안정성으로 보이며 우리 쪽에서 더 줄이기는 어려운 영역입니다.

### 4. 전략과 무관한 기존 보유종목 발견
