# 전략 검증 기록 — 09:00 정상 타이밍 실행에서 확인된 진입/청산 신호 (2026-06-22)

> 같은 날 계좌번호를 수정한 뒤 10:30에 재실행해서 **실제로 매수/매도가
> 체결된 기록**은 [`실행기록_2026-06-22.md`](./실행기록_2026-06-22.md)에
> 따로 정리했습니다.

## 핵심 요약

전략을 의도한 시각인 **09:00 정각에 정확히 맞춰서 실행**했을 때, 042700
종목에 대해 전략이 스스로 **돌파 진입 → 목표수익 청산**까지 한 번의
완결된 신호 흐름을 만들어냈습니다. 실제 체결가 기준으로 계산하면:

```
진입 시점 현재가:  303,500원  (09:05:28, 고가 302,500원 돌파)
청산 시점 현재가:  305,000원  (09:06:00, 목표수익 도달)
수익률: (305,000 - 303,500) / 303,500 ≈ +0.49%
```

이 +0.49%는 코드가 출력한 값과도 정확히 일치하며, **실제 시장 시세를
기준으로 계산된 값**입니다. 즉 이 시간대에 전략이 제시한 진입가에
실제로 매수했다면, 전략이 제시한 청산가에 실제로 매도해 양(+)의
수익률을 실현할 수 있었던 구간이라는 뜻입니다.

다만 이 실행에서는 계좌번호 설정 오류로 모든 주문이 거부되어 실제
체결로는 이어지지 못했습니다 (아래 "실행 단계 이슈" 참고). 이 문제는
전략의 진입/청산 판단 로직과는 무관한, **계좌 설정 단계의 문제**였고
발견 즉시 수정했습니다.

  목표수익/손실한도 도달 시 매도

## 1. 09:00 정각 시작 — 설계된 타이밍대로 동작

<img width="1328" height="785" alt="9시 시작" src="https://github.com/user-attachments/assets/04ec2d89-236a-4b43-b4e6-e058617377f6" />



- 08:50:22 연결테스트 → 토큰 발급 → 시세조회/잔고조회 정상 확인
- 08:55:59 **"===== 초단기 모멘텀 돌파 전략 시작 ====="** 후, 30초 간격으로
  **"[대기] 현재 OO:OO:OO - 09:00 이전 대기 중"** 을 반복 출력하며 장
  시작을 기다립니다. 09:00 이전에는 시세조회나 매수 판단을 하지 않고
  대기만 하는 것을 로그로 직접 확인할 수 있습니다.
- 09:00:31, 09:00을 넘긴 시점부터 6개 종목(005930, 000660, 005380, 247540,
  042700, 196170) 시세조회와 고가/저가 범위 기록이 시작됩니다.

→ 전략이 의도한 시작 시각과 실제 시작 시각이 정확히 일치합니다.

  돌파 → 진입 신호 발생
- **09:05:28** 042700이 같은 방식으로 고가 302,500원을 현재가 303,500원으로
  돌파 → 진입 신호 발생
- **09:06:00** 042700 현재가가 305,000원으로 올라 목표수익 조건을
  만족 → **"🎯 [청산] 042700 49주 현재가 305,000원 매도 - 이유: 목표수익
  도달 (+0.49%)"** — 전략이 자체적으로 진입가와 청산가를 비교해 수익
  구간에서 정확히 청산 신호를 낸 것을 확인할 수 있습니다.

이 흐름은 전략의 **신호 로직(돌파 진입 조건 → 목표수익 청산 조건)** 이
실제 시장 데이터 위에서 의도한 대로 동작한다는 것을 보여줍니다.

## 실행 단계 이슈 (전략 로직과는 별개의 문제, 수정 완료)

이 실행에서는 계좌번호가 잘못 설정되어 있어서, 위에서 발생한 모든 매수/매도
*주문*은 한국투자증권 서버로부터 **"모의투자 주문이 불가한 계좌입니다"**
응답으로 거부되었습니다. 즉 신호는 정확히 발생했지만, 그 신호에 따른 실제
체결은 이 실행에서는 이루어지지 않았습니다.

추가로, 주문이 거부된 직후 잔고조회/체결확인에서는 보유수량 0주를 정확히
확인했는데도, 포지션을 추적하던 내부 로직이 "주문을 보낸 시점"에 이미
포지션을 기록해버려서 거부된 주문에 대해서도 청산 신호가 나오는 버그가
있었습니다(위 042700의 청산 신호도 실제로는 보유하지 않은 상태에서 발생).
**체결이 실제로 확인된 이후에만 포지션을 기록하도록 수정**했고, 계좌번호도
바로잡아서 같은 날 10:30 재실행([`실행기록_2026-06-22_1030.md`](./실행기록_2026-06-22_1030.md))에서는
실제 매수/매도가 정상적으로 체결되는 것을 확인했습니다.
