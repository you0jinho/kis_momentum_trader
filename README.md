## 폴더 구조
```
kis_momentum_trader/
```

## 실행 방법

### 1. 가상환경 및 패키지 설치
```bash
python -m venv venv
.\venv\Scripts\Activate.ps1   # Windows (PowerShell)
# venv\Scripts\activate.bat   # Windows (cmd)
# source venv/bin/activate    # Mac/Linux
pip install -r requirements.txt
```

### 2. 환경변수 설정 (절대 코드에 직접 입력하지 말 것)

**방법 A — `.env` 파일 사용 (추천, 터미널 새로 열어도 매번 다시 입력 안 해도 됨)**

1. `.env.example` 파일을 복사해서 같은 폴더에 `.env` 라는 이름으로 저장
2. `.env` 파일을 열어 실제 값으로 채우기:
   ```
   GH_ACCOUNT=12345678-01
   GH_APPKEY=발급받은 모의투자 앱키
   GH_APPSECRET=발급받은 모의투자 앱시크릿
   ```
3. `.env`는 `.gitignore`에 등록되어 있어 git에는 올라가지 않음 — 그래도 다른 사람에게
   파일을 공유하거나 캡처할 때는 이 파일이 포함되지 않도록 주의할 것

`python main.py` 실행 시 `config.py`가 자동으로 `.env`를 읽어서 환경변수로 등록합니다.

**방법 B — 터미널에서 직접 설정 (.env 없이 그때그때)**

```bash
# Windows (PowerShell)
$env:GH_ACCOUNT="12345678-01"
$env:GH_APPKEY="발급받은 모의투자 앱키"
$env:GH_APPSECRET="발급받은 모의투자 앱시크릿"

# Mac/Linux
export GH_ACCOUNT="12345678-01"
export GH_APPKEY="발급받은 모의투자 앱키"
export GH_APPSECRET="발급받은 모의투자 앱시크릿"
```
이 방식은 터미널을 닫으면 값이 사라져 다시 입력해야 합니다. 터미널에 이미 이
방식으로 값이 설정되어 있으면 `.env` 파일보다 우선 적용됩니다.

### 3. 잔고/연결 확인 (실행 전 추천)
거래 시간(09:00~09:30)과 무관하게, 지금 바로 인증·시세조회·잔고조회가 되는지
확인할 수 있습니다. API 호출은 시세조회 1회 + 잔고조회 1회뿐이라 호출량
걱정 없이 매번 켤 때마다 먼저 돌려봐도 됩니다.
```bash
python test_connection.py
```
출력 중 아래 줄에서 현재 가용 현금(주문 가능 금액)을 확인할 수 있습니다.
```
[연결테스트] 잔고조회 성공 - 보유종목: {...} / 가용현금: 00,000,000원
```

### 4. 실행
```bash
python main.py
```
09:00 이전에 켜두면 대기 상태로 있다가 09:00부터 자동으로 동작을 시작합니다.
09:30이 되면 보유 중인 포지션을 모두 정리하고 프로그램이 스스로 종료됩니다.

## ⚠️ 실행 전 반드시 확인할 것
이 코드는 모의투자 전용으로 작성되었고, 일부 tr_id와 응답 필드명은 제가 100%
확신하지 못해 `config.py`의 `TrId` 클래스와 각 모듈 주석에 "확인 필요"로
표시해두었습니다. `test_connection.py`로 실제 테스트해본 결과는 아래와 같습니다.

- ✅ **확인됨** (실제 호출 성공) — `TrId.CURRENT_PRICE`(시세조회), `TrId.BALANCE`
  (잔고조회) 및 관련 필드명(`stck_prpr`, `hldg_qty`, `dnca_tot_amt`, `pdno`)
- ⚠️ **아직 미확인** (실제 주문을 넣어본 적 없음) — `TrId.BUY_ORDER`,
  `TrId.SELL_ORDER`, `auth.py`의 `get_hashkey()` 요청 경로

처음 매수/매도가 실제로 들어갈 때 로그(`logs/trader.log`)를 꼭 확인해서
주문 응답이 정상인지 보세요. 참고 자료: 한국투자증권 공식 GitHub
`koreainvestment/open-trading-api` 의 `examples_user/domestic_stock/`
샘플 코드와 비교해보면 빠르게 확인할 수 있습니다.

## 호출량을 줄인 설계 포인트
- 토큰은 당일 1회만 발급하고 `token_cache.json`에 캐시해서 재사용
- 09:00~09:05 구간은 20초, 09:05~09:30 구간은 18초 간격으로 폴링 (종목이
  늘어난 만큼 호출 폭증을 막기 위해 기존보다 늘림, `config.py`에서 조정 가능)
- 잔고조회는 "자금 배정 시 1회" + "주문 1건당 체결확인 1회"로만 호출
- 진입/청산 모두 시장가 주문 → 별도의 정정/취소 호출이 필요 없음
- 가용 현금은 종목 수로 균등 분배 (`CASH_SAFETY_MARGIN`로 10% 여유분 확보) →
  종목을 추가/제거해도 자금 배분 로직을 따로 안 고쳐도 됨

## 주의사항
- 모의투자 전용으로 설계되었습니다. 실전 도메인으로 전환하려면 `config.py`의
  `BASE_URL`과 모든 tr_id를 실전용 값으로 바꿔야 합니다 (이 프로젝트는 그
  전환을 권장하거나 지원하지 않습니다).
- 3분 타임컷과 ±0.3% 판정은 폴링 간격(`TRADE_POLL_SEC`, 기본 18초) 단위로
  점검되므로 초 단위로 정확하지는 않습니다. 더 정밀하게 하려면 이 값을
  줄이되, 모의투자 호출 제한에 걸리지 않는지 확인하세요.
- 모의투자 서버가 간헐적으로 500 Internal Server Error를 반환하는 경우가
  있습니다. `api_client.py`의 재시도 로직(최대 2회, 2초 간격)이 자동으로
  복구를 시도하니, 로그에 에러가 한 번 찍혔다가 바로 다음 줄에서 성공하는
  정도는 정상입니다. 같은 호출이 반복적으로 계속 실패할 때만 실제 문제로
  보고 점검하세요.
